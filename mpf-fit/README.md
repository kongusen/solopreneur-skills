`mpf-fit` 是一个 Market-Product-Founder Fit 对话决策 skill。  
它不帮你“热血上头”，而是帮你把机会拆清楚：市场是否真实、产品是否可行、你是否适配、下一步如何低成本验证。

你可以把它用于：

- 赛道分析
- 机会评估
- 要不要做某个项目
- 一人公司/副业方向选择
- 已决定要做后的胜率优化

## 效果示例

用户  
❯ 我在纠结做 AI 选题工具还是做简历优化 SaaS，哪个更值得做？

mpf-fit  
❯ 先不急着二选一。先做两步澄清：  
1）这两个方向里，哪个问题你有真实一线观察？  
2）如果 6 个月没有收入，你更愿意长期做哪一个？  

先回答这两点，我再带你做第一轮对比。

---

用户  
❯ 我决定做了，但担心获客太难。

mpf-fit  
❯ 那我们切到 C 模式（提升胜率）。  
先只看一个维度：可触达性。  
你最可能拿到第一批 20 个目标用户的地方，是哪个具体社区或渠道？

---

用户  
❯ 出报告

mpf-fit  
❯ 可以。我会基于现有信息输出快速版报告；  
如果你要“明确建议（Go/No-Go）”，我再追加决策版。


## 它分析什么

`mpf-fit` 不是单一问答模板，而是“主控 + 模块”结构：

- **Market（市场）**：需求真实性、市场规模、竞争格局、可触达性、时机窗口
- **Product（产品）**：解法有效性、MVP 路径、差异化、复杂度可控性
- **Founder（个人）**：能力匹配、竞争优势、动机持续性、资源禀赋、机会成本
- **Validation（验证）**：一周内最小实验、付费验证、首批用户来源
- **Risk（风险）**：市场风险、产品风险、增长风险、个人风险

它会每轮只推进一个维度，避免信息轰炸。

## 工作原理

输入一个机会后，`mpf-fit` 做四件事：

1. **两问启动**：先判断你处于 A（选方向）/B（评估机会）/C（提升胜率）哪种阶段。  
2. **模块路由**：按“信息缺口最大 + 决策影响最大 + 用户当前关注”选择下一轮模块。  
3. **状态记忆**：内部维护结构化状态（market/product/founder/validation/risk），支持多轮累积。  
4. **模板出报告**：用户说“出报告”后，按快速版/完整版/建议版模板生成。

---

## 诚实边界

这个 skill 会明确指出“不确定性”，而不是装作全知：

- 信息不足时，不给伪结论
- 没有付费信号时，不把“有人点赞”当“有人付钱”
- 用户已决定要做时，聚焦“提高胜率”，不做情绪化否定
