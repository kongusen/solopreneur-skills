# 产品解法分析模块

## 触发条件
当需要判断“这个方案是否可行且可做”时使用。

## 核心问题
- 你的方案是否真正解决核心问题，而不是只优化表面流程？
- 最小可验证版本（MVP）是什么？多久能做出来？
- 用户为什么会选择你而不是现有替代方案？
- 技术与运营复杂度是否超出个人可承受范围？

## 深挖路径
问题定义 -> 解法机制 -> MVP 范围 -> 差异化 -> 复杂度评估

## 输出格式
- 解法有效性：高 / 中 / 低
- MVP 定义：一句话
- 关键复杂度风险：1 条
# 产品解法分析模块

## 触发条件
当需要判断“这个想法是否做得出来、做出来是否有价值”时使用。

## 核心问题
- 你的方案相比替代方案具体好在哪里？
- 最小可验证版本（MVP）是什么，多久可上线？
- 技术与运营复杂度是否适合当前资源？

## 深挖路径
问题定义 -> 解法设计 -> MVP 切分 -> 差异化 -> 复杂度评估

## 输出格式
- 解法有效性：强 / 中 / 弱
- MVP 定义：一句话
- 主要复杂度风险：1 条
