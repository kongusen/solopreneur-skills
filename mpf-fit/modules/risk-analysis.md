# 风险拆解模块

## 触发条件
当需要识别盲点、降低失败概率或做取舍时使用。

## 风险维度
- 市场风险：需求不存在，或付费意愿不足
- 产品风险：做不出来，或交付质量不达标
- 增长风险：获客成本高、转化路径不成立
- 个人风险：现金流、时间、精力不可持续

## 深挖路径
风险识别 -> 影响评估 -> 预警信号 -> 缓释动作

## 输出格式
- Top 3 风险：按优先级
- 每个风险的预警信号：1 条
- 每个风险的缓释动作：1 条
# 风险拆解模块

## 触发条件
当用户需要做 Go / No-Go 判断，或存在明显不确定性时使用。

## 风险维度
- 市场风险：需求不足或付费意愿弱
- 产品风险：难以在可接受时间内交付可用产品
- 增长风险：获客与转化成本过高
- 个人风险：时间、现金流、精力不可持续

## 核心问题
- 哪个风险最可能在 3 个月内导致失败？
- 哪个风险能通过小实验优先验证？

## 输出格式
- Top 3 风险（按影响与概率排序）
- 每项风险对应 1 条缓释动作
